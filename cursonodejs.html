Comandos básicos

Usaremos la consola de GitHub para ejecutar los comandos directamente desde la carpeta de origen:

node sirve para ejecutar cualquier archivo en el servidor.

node init inicializa un nuevo proyecto en la carpeta en la que estamos trabajando.

npm i o npm install sirven para instalar paquetes (librerías) en el proyecto. Se puede poner el nombre del paquete a continuación o meterlos en el apartado dependences del json (Javascript Object Notation) que se genera al inicializar el proyecto. Si añadimos --g al final del comando, el paquete se instala en el sistema operativo en lugar de en el proyecto, permitiendo usarlo desde cualquier proyecto.

npm start arranca el proyecto desde el que hayamos ejecutado la consola. Accederemos a la aplicación escribiendo localhost:3000 en la barra del navegador (3000 es el puerto por defecto, que se puede consultar en el archivo www de la carpeta bin).

Paquetes instalados

npm install express-generator --g. Lo usaremos con express amor --hbs para crear un proyecto llamado amor (el --hbs indica que hay librerías que hay que instalar con npm install.

npm install --save nodemon. Sirve para reiniciar el servidor cada vez que guardemos un cambio en un archivo, así no hay que reiniciarlo manualmente.
